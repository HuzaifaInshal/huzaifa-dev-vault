# API Integration

Two-step workflow for integrating REST APIs. Start with the **base prompt** to establish the service patterns, then send **later prompts** per feature with the actual API specs.

## How it works

1. Send the **Base Prompt** once at the beginning of a session (or add it as a project-level instruction/agent doc)
2. For each feature, follow up with a **Later Prompt** listing the API endpoints, the target views, and any relevant payload shapes
3. The AI generates typed service files, query/mutation hooks, and wires them into the correct views — all following your project structure

## Service Patterns Covered

| Pattern | Hook | Use when |
|---------|------|----------|
| GET (paginated) | `useQuery` | Fetching lists or detail data |
| POST / PUT / PATCH / DELETE | `useMutation` | Creating or mutating data |
| Infinite scroll | `useInfiniteQuery` | Paginated lists with load-more |
