# cn

Merges Tailwind CSS class names without conflicts. Combines `clsx` for conditionals with `tailwind-merge` to resolve Tailwind's specificity edge cases.

## Source

<CodeBlock source={cnSrc} language="ts" title="src/utils/cn.ts" />

## Usage

```ts
import { cn } from '../utils/cn'

// Conditional classes
cn('px-4 py-2', isActive && 'bg-violet-500')

// Tailwind conflict resolution — later class wins
cn('px-4', 'px-2')
// → 'px-2'  (twMerge resolves the conflict)

// Both together
cn('text-sm px-4', isLarge && 'text-lg px-6', className)
```

## Why not just clsx?

`clsx` concatenates class strings but doesn't resolve Tailwind conflicts. If you pass `px-4` and `px-2`, both end up in the output and the winner depends on stylesheet order — which is unpredictable. `twMerge` removes the earlier conflicting class, so the last one always wins.
